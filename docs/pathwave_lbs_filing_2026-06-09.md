# PathWave — 위치기반서비스 사업 신고 가이드

> **작성일**: 2026-06-09
> **대상**: 위치정보의 보호 및 이용 등에 관한 법률 (위치정보법) §9 — 위치기반서비스 사업 신고
> **운영사**: 주식회사 트리거소프트 (TRIGGER SOFT CO., LTD.)
> **신청 경로**: [방송통신위원회 전자민원](https://emnet.kcc.go.kr) 또는 [정부24](https://www.gov.kr)
> **비용**: 무료
> **처리 기간**: 2~4주

---

## 0. 신고 의무 근거

### 0-1. 사업 분류
PathWave 가 BLE 비콘 + GPS 좌표를 활용하여 매장 검색·자동 연결을 제공 → **위치기반서비스 사업자** 해당.

### 0-2. 두 가지 종류

| 분류 | 대상 | PathWave |
|---|---|---|
| 위치정보**사업자** | 개인위치정보 수집·이용·제공자 (예: 통신사, GPS 단말 제조사) | ❌ 해당 안 됨 |
| **위치기반서비스 사업자** | 타사 위치정보 또는 비개인 위치정보 활용 서비스 운영 | ✅ **해당** |

→ **"위치기반서비스 사업자 신고"** 만 진행하면 됨 (위치정보사업 허가는 불필요).

### 0-3. 미신고 시 처벌
- 위치정보법 §40 — 시정명령 + 과태료 3천만 원 이하
- 출시 전 신고증 수령 필수

## 1. 신청 흐름 한눈에

```
[방송통신위원회 전자민원 (또는 정부24) 로그인]
   ↓
[위치기반서비스 사업 신고서 작성]
   ↓
[첨부 서류 6종 업로드]
   ↓
[제출 → 심사 2~4주]
   ↓
[신고필증 발급 → admin-web 법인 정보에 신고번호 입력]
   ↓
[3 콘솔 footer + 약관 8종 자동 동기]
```

| 항목 | 입력값 | 비고 |
|---|---|---|
| 법인명 | **주식회사 트리거소프트** | 영문: `TRIGGER SOFT CO., LTD.` |
| 사업자등록번호 | (10자리) | ⚠️ 사용자 입력 |

| 본점 소재지 | (창업보육센터 주소) | ⚠️ |
| 대표 전화 / 이메일 | (사무실 전화 / support@pathwave.???) | ⚠️ |
| 통신판매업 신고번호 | (통판 신고증 발급 후 입력) | 통판 가이드 참조 |

## 3. 입력값 — 서비스 개요

| 항목 | 입력값 |
|---|---|
| 서비스명 | **PathWave** |
| 서비스 분류 | **생활·편의** — 매장 검색·근거리 인증 |
| 주요 기능 | 1) BLE 비콘 감지 → 매장 식별 → WiFi 자동 연결<br>2) 거리 기반 매장 검색 (사용자 GPS 좌표 일시 활용)<br>3) 매장별 스탬프·쿠폰 자동 적립 (위치 트리거 부분) |
| 서비스 시작 예정일 | **2026.08.01** (또는 정식 출시일) |
| 서비스 제공 방식 | iOS / Android 모바일 앱 + 웹 (시설관리자 / 운영자 콘솔) |
| 서비스 지역 | 대한민국 전국 |

---

## 4. 입력값 — 위치정보 처리 항목

| 항목 | 처리 여부 | 비고 |
|---|---|---|
| 개인위치정보 (실시간 사용자 좌표) | **수집** (서비스 동의 후) | 매장 검색·BLE 매칭에 활용 |
| 비개인 위치정보 (매장 좌표) | **수집** | 시설관리자가 매장 등록 시 입력 |
| 위치정보 저장 | **30일 후 익명화** (user_wifi_logs 기준) | 익명화 = user_id 해시 마스킹 |
| 제3자 제공 | **없음** | 외부 광고/제휴 제공 X |
| 국외 이전 | **없음 (1차)** 또는 **싱가포르 (Contabo)** | 사용자 결정에 따라 |

⚠️ Contabo 싱가포르 서버 사용 시 = **국외 이전 있음** 으로 표기 + 약관에 국외 이전 항목 명시.

---

## 5. 입력값 — 위치정보 보호 조치 (※ 핵심)

### 5-1. 기술적 보호조치

| 항목 | 적용 |
|---|---|
| 전송 구간 암호화 | TLS 1.2+ (HTTPS 전 구간 강제) |
| 저장 데이터 암호화 | AES-256 (WiFi 비밀번호·PG 빌링키). 위치 로그는 익명화 |
| 접근 통제 | JWT HS256 + sub_type 검증 + 슈퍼어드민 별도 권한 |
| 비밀번호 보호 | bcrypt 해시 (평문 저장 금지) |
| 로그 보관 | gunicorn access log + Sentry 에러 추적 — 90일 보관 |
| 보안 패치 | 운영 단계 분기별 점검 |

### 5-2. 관리적 보호조치

| 항목 | 적용 |
|---|---|
| 책임자 지정 | **대표이사** (위치정보 관리·보호 책임자 겸 개인정보 보호책임자) |
| 직원 교육 | 1인 회사 — 대표이사 자체 학습 (정기) |
| 접근 권한 관리 | 슈퍼어드민 = 대표이사 1인. 향후 직원 추가 시 권한 분리 |
| 침해사고 대응 | KISA 침해사고 24시간 신고 절차 명시 |
| 정기 점검 | 분기 1회 보안 점검 보고서 작성 |

### 5-3. 물리적 보호조치

| 항목 | 적용 |
|---|---|
| 서버 보관 | Contabo (싱가포르) 데이터센터 — IDC 24h 보안 / 향후 한국 리전 추가 검토 |
| 사무실 출입 | 창업보육센터 — 보안카드 + CCTV |
| 단말 분실 | 노트북 디스크 암호화 (FileVault / BitLocker) |

## 6. 첨부 서류 6종 체크리스트

| # | 항목 | 발급처 | 비고 |
|---|---|---|---|
| 1 | 사업자등록증 사본 | 홈택스 | 보유 |
| 2 | 법인등기부등본 (3개월 이내) | 인터넷등기소 | ₩1,000/통 |
| 3 | **위치정보 관리·보호 책임자 지정서** | 자체 작성 | 대표이사 본인 지정 — 양식 1쪽 |
| 4 | **위치정보 처리방침** | 자체 작성 | PathWave 약관 8종 중 1종 (이미 준비) — KCC 양식 매핑 |
| 5 | **기술적·관리적 보호조치 명세서** | 자체 작성 | 본 가이드 §5 내용 그대로 정리 |
| 6 | **서비스 개요서** | 자체 작성 | "BLE 비콘 기반 WiFi 자동 연결 서비스" + 본 가이드 §3 내용 |

⚠️ 4·5·6은 형식 자유 — A4 1~3장 워드 문서로 작성.

## 7. 사용자 입력 필요 항목 종합 (⚠️)

신고서 작성 전 손에 두기:

1. ⚠️ 사업자등록번호 10자리
2. ⚠️ 법인등록번호 13자리
3. ⚠️ 대표자 성명
4. ⚠️ 본점 소재지 (전체 주소)
5. ⚠️ 대표 전화 + 이메일
6. ⚠️ 통신판매업 신고번호 (먼저 통판 신고 후)
7. ⚠️ **서비스 시작 예정일** 확정
8. ⚠️ **국외 이전 여부** 결정 (Contabo 싱가포르 그대로 / 한국 리전 추가)
9. ⚠️ 서버 보관 위치 (싱가포르 / 한국)
10. ⚠️ 첨부 서류 3·4·5·6 작성 (각 A4 1~3장)

## 8. 신고증 발급 후 PathWave 작업

| # | 작업 | 위치 |
|---|---|---|
| 1 | 위치기반서비스 사업 신고번호 입력 | admin-web → 시스템 → 법인 정보 |

| 5 | Apple App Store / Google Play 심사 자료 첨부 | 메타데이터 + Privacy Manifest |

## 9. KISA 침해사고 신고 (사고 시)

위치정보 침해사고 발생 시 24시간 내 KISA 신고 의무:

- **신고 채널**: ☎ 118 / [KISA 침해사고 신고](https://www.krcert.or.kr)
- **신고 사항**: 사고 일시, 사고 유형, 피해 규모, 임시 조치 사항
- PathWave 운영 매뉴얼에 KISA 신고 절차 포함 (수행사 인계용 매뉴얼에 명시)
