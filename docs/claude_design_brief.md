# PathWave — Claude Design 온보딩 브리프

> **용도**: claude.ai/design 온보딩 시 이 저장소를 읽힐 때, 또는 첫 프롬프트로
> 붙여넣어 Claude Design 에게 "무엇을·어떻게·무엇은 절대 건드리지 말지"를 알리는 문서.
> 대상은 **mobile (Flutter) 앱**. provider-web / admin-web 은 이번 범위 아님.

---

## 0. 한 줄 요약

> PathWave mobile 의 **다크 디자인을 출시 수준으로 끌어올린다.**
> 색·폰트·토큰 정책은 **그대로 두고**, 빈/에러/로딩 상태와 정보 위계·여백 리듬을
> DesignCode 풍 모던 다크로 다듬는다.

## 1. 먼저 읽을 진입점 (코드)

| 파일 | 내용 |
|---|---|
| `mobile/lib/utils/app_theme.dart` | **디자인 토큰 단일 소스** (색·라운드·여백·그림자·타이포). 모든 값의 출처. |
| `mobile/lib/widgets/pw.dart` + `pw_*.dart` | 공통 컴포넌트(`Pw*`). 신규 디자인은 이걸 재사용/확장. |
| `docs/DESIGN_SYSTEM.md` §10 | mobile 토큰 규칙 + 화면 인벤토리(26개) + 컴포넌트 props. |

## 2. 🔒 절대 제약 (변경 금지 — 어기면 핸드오프 거부)

1. **색 정책 고정**. mobile 포인트색 = **보라 `#7C3AED`** (확정 정책). 임의로 색을
   바꾸지 말 것. 3콘솔 색 정책: mobile=보라 / provider-web=녹색 `#22C55E` /
   admin-web=블루 `#2563EB`. (PathWave 브랜드 일관성의 핵심)
2. **다크 모드 전용**. 라이트 테마 도입 금지. 배경 `#0F0F1A`, 카드 `#1E1E2E`.
3. **OS 시스템 폰트** 사용 (iOS=SF / Android=Roboto). 커스텀 폰트 번들 금지.
4. **i18n 10개 언어** 노출. ko·en·zh-CN·ja·zh-TW·vi·th·tl·id·ms. 텍스트 길이가
   언어마다 다르므로 **고정폭·1줄 가정 금지**(말줄임/줄바꿈 허용 레이아웃).
5. **토큰만 사용**. 새 디자인도 `AppTheme` 토큰(rSm/rMd/rLg·s1~s6·softShadow·
   textTheme)으로 표현. 하드코딩 숫자/hex 금지 (한 곳에서 일괄 변경 가능해야 함).
6. **컴포넌트 재사용**. 새 변형은 `Pw*` 위젯을 확장 — 화면마다 일회성 위젯 양산 금지.

## 3. 🎯 개선 목표 (우선순위 — 위에서부터)

| 순위 | 항목 | 왜 |
|---|---|---|
| **2-a** | **빈 / 에러 / 로딩 상태** (`PwEmptyState`·`PwErrorState`) 모던화 | **10개 화면이 한 번에** 좋아짐. 현재 아이콘+텍스트만 덩그러니. 일러스트·여백 리듬·명확한 CTA 추가 |
| 2-b | 정보 위계 — 카드 강약(주/보조), 섹션 헤더 | 지금은 같은 톤 카드 나열로 평면적 |
| 2-c | 여백 리듬 + soft shadow depth | DesignCode 톤의 "공기감" |
| 2-d | 핵심 화면 디테일 — 홈 / 매장(facility) / WiFi 연결 / 회원 QR | 사용 빈도 최상위 |

> ⚠️ `home_screen` 에 **세로 오버플로우 버그(211px)** 존재 — 디자인과 별개로 코드 수정 필요(핸드오프 받는 쪽에서 처리).

## 4. 🎨 톤 & 방향

- **레퍼런스**: DesignCode UI Kit 풍 모던 다크 — 둥근 카드(`rLg` 20), 미묘한
  soft shadow, 넉넉한 여백, 큰 타이틀 + 작은 보조텍스트의 명확한 대비.
- **빈 상태**: "비어 있어도 완성도 있게" — 부드러운 일러스트/아이콘 + 안내 +
  다음 행동 CTA (예: 쿠폰 없음 → "주변 매장 둘러보기").
- **에러 상태**: 기술 용어 금지(이미 `friendlyError` 로 메시지 정제됨) + 재시도 버튼.
- **로딩**: 빈 화면 대신 스켈레톤/플레이스홀더 지향.

## 5. 🔁 핸드오프 규칙 (→ Claude Code)

디자인이 확정되면 Claude Design 의 **handoff bundle** 을 만들어 Claude Code(이 저장소
작업자)에게 전달한다. 받는 쪽은:

1. 디자인의 색·여백·라운드를 **`AppTheme` 토큰으로 매핑**(새 값이 필요하면 토큰부터 추가).
2. `Pw*` 컴포넌트 수정/추가로 구현 — 화면 코드 변경 최소화.
3. iPhone 17 Pro 시뮬레이터로 **시각 확인** 후 그룹별 PR.

> 즉 Claude Design 은 **"무엇을"**(디자인 의도), Claude Code 는 **"어떻게"**(토큰·
> 컴포넌트 구현)를 맡는다. 색 정책·토큰 구조는 구현 단계에서 강제된다.

## 6. 현재 상태 (참고)

- 1단계 완료: 토큰 단일소스(`AppTheme`) + `PwCard`/`PwButton` 토큰화 (#263).
- raw 예외 노출 19화면 정리 (#264) — 에러 메시지는 이미 사용자 친화적.
- 2단계(하드코딩 274건 토큰 치환)는 점진 진행 중 → Claude Design 디자인과 함께 정리 가능.
