# PathWave — C-3 페르소나 통합 테스트 시나리오

> **작성일**: 2026-05-23
> **단계**: 출시 5단계 중 **1단계** (로컬개발 + 테스트데이터 + stub 환경) 마지막 검증
> **목적**: 페르소나별 end-to-end 흐름을 실제 사용자가 쓰는 순서대로 검증하고, **API 는 있는데 UI 없음 / 식별자 없음 / 흐름 불일치** 같은 인접 구멍을 출시 전 마지막으로 잡아낸다.
> **검증 환경**: stub (Firebase, DeepL, SendGrid, 결제) + sqlite 로컬 DB + 더미 비콘 (DB row).
> **실 키/실 비콘 검증은 5단계 중 3단계** (외부 서비스 신청 → 소스 + 서버 업데이트 후).

## 7. Pass / Fail / Hole 기준

- **PASS** — 시나리오 끝까지 진행, 검증 포인트 모두 만족, 인접 구멍 없음.
- **FAIL** — 흐름 도중 에러/거부/하드코딩으로 막힘. 즉시 수정 PR.
- **HOLE** — 흐름은 통과하지만 인접 구멍 (UI 없음/i18n 누락/안내 부족) 발견. 우선순위 매겨 별도 큐.
