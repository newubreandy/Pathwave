**v1에서 새로/마저 할 것** — 워크스트림 W1~W7 (§6)

### 🟡 2차 — 출시 직후 업데이트

- 추가 언어·국가 (협의 후)
- 메뉴 자동 번역 (한식 용어 사전 기반 — OCR+자동번역만으론 불가, 사전 필요)
- WiFi 로밍 C: 웹 캡티브 포털(노트북), `radius` 모드, 꺼진 앱 깨우기
- 고객응대 CMS 자동화 Stage 1 (카톡 챗봇 · 이메일 AI)

### 🔵 3차 — 확장

- 글로벌 멀티리전 인프라 + 국가별 패키징 + 통합 어드민 검색·리포팅
- 자동화 Stage 2/3, Eddystone-EID 회전 비콘, POS 제휴·B2B 확장

### ⛔ 보류 / 제외

- 외국인 결제(알리페이·위챗페이) · 면세 자동 — 외부 계약 필요, 현재 비현실적. 추후 재검토.

## 3. 언어 모델

- **사용자앱**: UI = 디바이스 설정 언어. 미지원 → **영어** fallback.
- **매장주(provider)**: 한국어 (한국인 전용). 추후 모바일/PC 접근도 디바이스 언어 동일 원리.
- **지원 언어 셋**: Google 번역 품질 테스트(≥90%) 통과 언어만 제공.
- **중국어 = 3개**: 간체(zh-CN) / 번체·대만(zh-TW) / 홍콩(zh-HK).
- i18n 인프라(translations DB·API)는 구현됨(목업) → 실연동·언어 채우기·UI 검증 필요.
- 채팅 번역 엔진(`models/translator.py`, Google API 프로바이더)은 구현됨 → **채팅에 연결** 필요.

## 4. 품질 기준 (모든 작업에 강제)

1. 디자인 시스템 강제 — 공용 컴포넌트(Pw*)·디자인 토큰만
2. Apple HIG + Material 3 체크리스트 준수
3. Apple/Google 심의 가이드 디테일까지
4. 3콘솔 연계 수정 (개별 콘솔 단독 수정 금지)
5. 유사 서비스(카카오/토스/배민/네이버/당근) 벤치마크 선행
6. 답변 전 항상 코드/사실 확인 — 추측 금지
7. "완료" 전 시나리오 테스트 — 사용자 관점 + 심의 심사관 관점 → 결과 보고
