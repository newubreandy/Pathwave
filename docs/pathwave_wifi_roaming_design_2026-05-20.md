# PathWave 무중단 WiFi 로밍 — 설계 및 우선순위 정리

- 문서 버전: 2026-05-20
- 상태: 확정 검토용 (검토 후 구현 착수)
- 회사: 주식회사 트리거소프트 (법인등록번호 214911-0008439)

---

## 1. 문서 목적

PathWave 핵심 기능 "무중단 WiFi 로밍"의 설계를 한 곳에 정리한다. 즉시 착수할 부분과
이후 확장할 부분을 우선순위로 나눠, 이 문서를 확정하면 바로 구현 단위로 쪼갤 수 있도록 한다.

---

## 2. 핵심 서비스 정의

PathWave의 본질 = **신호 기반 무중단 WiFi 로밍**.

사용자가 PathWave가 제공하는 WiFi에 한 번 인증하면, 넓은 공간(리조트·호텔 등)을 이동해도
재인증 없이 가장 가까운 PathWave WiFi로 끊김 없이 연결된다. 비콘·스탬프·쿠폰·외국인
응대는 부가 트리거이며, 제품의 본질은 무중단 WiFi 경험이다.

예: 리조트 투숙객이 로비에서 1회 인증 → 객실·수영장·카페로 이동해도 계속 연결.

---

## 3. 현재 구현 상태 (격차)

현재 코드는 "비콘 1개 → WiFi 1회 연결" 수준이다.

- `handshake` API가 매장 WiFi를 1개만(LIMIT 1) 반환
- 다중 AP·신호 비교·자동 전환(핸드오프)·백그라운드 모니터링 없음
- 슈퍼어드민(admin-web)에 WiFi 등록 화면 없음
- 점주(provider-web) WifiSettings는 mock 데이터, 실 API 미연동
- 비콘 등록(admin-web)은 이미 구현됨 — 실기기 검증만 남음

---

## 4. 기술 제약 (요약)

### iOS (제약 큼)
- 앱은 주변 WiFi 스캔·신호강도 조회 불가 (공개 API 없음)
- WiFi 가입 시마다 시스템 팝업 발생 — 앱이 끌 수 없음
- → "WiFi 신호 기반 앱 자동 전환"은 iOS에서 불가능
- 가능한 것: 비콘 신호(블루투스)로 위치 판단 / 구성 프로파일로 다건 WiFi 1회 설치 /
  꺼진 앱도 비콘 구역 진입 시 깨어나 알림 발송

### Android (상대적으로 자유로움)
- WiFi 스캔·자동 연결 가능, 다건 등록 후 1회 승인 → 이후 자동

### 결론
- iOS는 "WiFi 신호 추적"이 아니라 "비콘 위치 판단 + 구성 프로파일"로 구현
- 진짜 무중단(팝업 0)은 단일 SSID 매장에서만. 호실별 다른 SSID 매장은
  "구성 프로파일 1회 설치 → 이후 매장 내 무중단"

## 5. 확정 아키텍처

### 5.1 비콘 ↔ WiFi = N:N
WiFi 1개에 비콘 여러 개, 비콘 1개가 WiFi 여러 개와 연관 가능. 연결 테이블 `beacon_wifi`로
모델링하여 1:1·1:N·N:N을 모두 표현.

### 5.2 WiFi 자격증명 관리 모드 (`credential_mode`)
PathWave는 남의 공유기 비밀번호를 원격 자동 변경 불가(라우터 제어권 필요). 3가지 모드를
매장이 선택:
- `static` — 관리자가 입력한 고정 비번 그대로 (기본)
- `managed` — 비번 교체는 관리자가, PathWave가 리마인드 + 인가 손님에게만 전파
- `radius` — WPA2-Enterprise + RADIUS, 손님별 개별 자격증명 (확장)

### 5.3 대량 등록 — iOS 구성 프로파일
WiFi 수십~수백 개를 파일 1개(.mobileconfig)에 담아 1회 설치. 서버가 매장+손님 객실
범위로 프로파일 생성, 외부 CA 발급 인증서로 서명. (1건씩 등록하는 방식은 50개=50팝업이라
대량에서 폐기.)

### 5.4 가맹점 2종류
- 단일 SSID 체인(스타벅스형): WiFi 1개 등록으로 전 지점 무인증 자동
- 호실별 다른 SSID(리조트형): 매장당 프로파일 1회 설치, 이후 매장 내 무중단

### 5.5 기간제 공간 격리
- `units` 테이블 (호실 / 독서실 자리 / 주차구역 등)
- WiFi에 `scope`(public 공용 / unit 개별공간) + `unit_id`
- `wifi_access_grant` — 모든 접근을 권한 1개로 통일
  (user, 대상, 시작일시, 종료일시, 출처[예약/결제/멤버십/무료])
- 손님별 프로파일 = [매장 공용] + [본인 grant 공간]만 → 타 공간 자동 격리
- 기간 종료: 앱이 만료 프로파일 제거 + 위치(비콘)·기간(grant) 둘 다 만족해야 연결

### 5.6 손님 동의·연결 모델
- 손님이 1회 인앱 동의 → 그 매장 WiFi 묶음을 체크인 시점에 일괄 등록
- 설정 선택지: "전체 자동 이용" / "매장·위치마다 승인"
- 점주가 WiFi 변경 시 → 앱이 불일치 감지 → "WiFi 정보가 변경되었습니다 [적용하기]"
- 꺼진 앱: 비콘 구역 진입 시 깨어나 로컬 알림 → 손님이 탭하면 진입 (**v1 미포함 확정** — 확장 C)

### 5.7 저장·확장성
- 폰은 PathWave 전체를 받지 않음 — 비콘이 현재 매장 식별 → 그 매장 WiFi 묶음만 fetch
- 한 매장 = 수~수십 KB → 앱 가벼움, 국가 크기 무관
- 서버 DB = 원본(점주 관리), 폰 = 현재/최근 매장 캐시 (앱 삭제 시 소멸)

### 5.8 보안
- 비콘은 위치 힌트일 뿐 — 비밀번호·사용자정보·서버접근 전무 → 해킹해도 유출 데이터 0,
  최악 피해 = 그 자리 서비스 정지(DoS)
- WiFi 정보는 항상 서버에서 TLS(인증서 고정)로만 전달
- 연결 후 검증: 공유기 BSSID + 서버 인증서 → 불일치 시 경고 + 연결 해제 (가짜 WiFi 차단)
- 공공 와이파이 위협(가로채기·피싱)은 실재 — 위 검증 + 앱 통신 암호화로 방어
- **OS 최소버전 정책 (신규)**: 앱 부팅 시 OS 버전 확인 → 지정 최소버전 미만이면 차단
  화면 + OS 업데이트 안내. 서버 설정값으로 조정. 미패치 OS 취약점 차단.
- 향후: Eddystone-EID 회전 ID로 비콘 복제 원천 차단

### 5.9 어드민 아키텍처
- 국가별 분리 ❌ → **통합 어드민 1개 + "국가" 차원**
- 화면·코드 1벌, 국가 드롭다운 필터, 전체 집계
- 권한: 본사 글로벌 관리자 = 전체 / 현지 관리자 = 자국만
- 데이터 저장은 법규(데이터 거주)에 따라 지역별 가능, 어드민은 하나

### 5.10 노트북·태블릿 등 비(非)앱 디바이스

- 태블릿(iPad / Android)은 동일한 모바일 앱이 그대로 구동 → 휴대폰과 동일하게 커버됨.
- 노트북(Windows / macOS) 등 앱이 없는 기기 = 핵심 원칙으로 해결: **이용 권한(grant)은
  기기가 아니라 "계정"에 귀속**한다.
  - 휴대폰 앱 = 프리미엄·무중단 클라이언트 (비콘 + 자동 연결)
  - 웹 캡티브 포털 = 범용 클라이언트. 브라우저에서 PathWave 계정 로그인 → 같은 grant
    확인 → 접속 허용. 노트북·앱 미설치 기기를 모두 커버.
- 노트북은 보통 한자리에 앉아 쓰므로 "무중단 로밍"보다 "한 번 쉽게 접속"이 핵심 → 웹
  포털로 충분. 별도 데스크톱 앱은 만들지 않는다(무설치 웹 포털이 더 낫다).
- 인프라 의존: 캡티브 포털 / 802.1X 는 `radius` 모드와 같은 게이트웨이가 필요 → RADIUS
  모드와 함께 확장. static 모드 매장에서는 노트북이 WiFi 비번을 일반 방식으로 사용
  (휴대폰 앱이 비번을 표시해 줄 수 있음).

### 5.11 계정·동시 사용 정책 (확정)

- **계정당 PathWave WiFi 동시 접속 = 1대만.** 한 계정을 여러 사람·여러 기기가 동시에
  쓰는 것을 차단한다 (카페 장기 이용자 등의 다인 공유·WiFi 무임승차 방지).
- "1대"는 동시 1연결을 뜻한다 — 기기는 휴대폰이든 노트북이든 무방하나 한 순간에 한 대.
  다른 기기가 접속하면 "기존 연결을 끊고 이 기기로 전환" 확인 후 전환한다.
- 휴대폰 앱은 로그인 시 계정-기기가 자동 연계(다른 폰 로그인 시 기존 폰 로그아웃).
  노트북·태블릿(웹 포털)도 동일한 1대 한도를 공유한다.
- 기간제 공간(grant)도 동일 — 자리·객실 1개 = 동시 1대.
- ⚠️ 기기 식별에 **MAC 주소는 쓸 수 없다** — iOS·Android 가 프라이버시 정책으로 앱의
  MAC 접근을 차단하고 WiFi MAC 을 랜덤화한다. 대신 앱 생성 기기 ID(iOS
  identifierForVendor 등, 보안 저장소 보관)를 사용한다. 앱 삭제·재설치 = 기기 교체로 처리.
- 이상 사용 감지(원거리 동시 접속 시도 등)는 확장(C).

## 6. 데이터 모델 (신규/변경)

| 테이블 | 구분 | 주요 필드 |
|---|---|---|
| `wifi_profiles` | 변경 | + scope, unit_id, credential_mode, bssid, country |
| `beacons` | 유지 | id, serial_no, uuid, major, minor, facility_id, status |
| `beacon_wifi` | 신규 | beacon_id, wifi_profile_id (N:N 연결) |
| `units` | 신규 | facility_id, name, type(room/seat/parking) |
| `wifi_access_grant` | 신규 | user_id, target_type, target_id, valid_from, valid_until, source |
| `devices` | 신규 | account_id, device_id(앱 생성 ID), device_label, platform, kind(app/portal), last_seen_at |

주요 API (신규/변경):
- `GET /api/beacon/handshake` — 매장 WiFi "묶음" 반환 (LIMIT 1 제거, grant 반영)
- `GET /api/wifi/venue/{facility_id}.mobileconfig` — iOS 구성 프로파일 생성·서명
- 슈퍼어드민 — WiFi / units / grant CRUD
- 점주 — WiFi CRUD 실연동
- `GET /api/version/min-supported` — min_os_ios / min_os_android 추가

## 7. 컴플라이언스 (개인정보 / 위치정보)

- WiFi 로밍은 위치 이력(언제 어느 매장·객실) = 민감 개인정보 취급
- 한국: 개인정보보호법(최소수집·암호화·보관기간) + 위치정보보호법(방통위
  위치기반서비스사업 신고 + 위치정보 별도 동의)
- 트리거소프트 정관 목적에 "Wi-Fi 서비스" "비콘·LBS 개발·운영업" 포함 → 정관 근거 확보
- 글로벌: GDPR(EU)/CCPA(미국) 등 국가별 → 국가별 패키지로 대응
- 방향: 데이터 최소화 + 가능한 한 폰 저장 + 암호화. 출시 전 전문가 검토 필수

## 9. ★ 우선순위 — 즉시 / 선행필요 / 확장

### A. 즉시 처리 가능 (외부 키 불필요 — 설계·코드로 지금)

1. 데이터 모델 전면 재설계 — `wifi_profiles` 확장 + `beacon_wifi` + `units`
   + `wifi_access_grant` + `devices`. **확장 필드(scope/unit/credential_mode/country)를 지금
   스키마에 전부 포함** (나중에 안 꼬이게)
2. `handshake` API 개선 — 매장 WiFi 묶음 반환
3. 슈퍼어드민 WiFi 등록 화면 신규 (admin-web)
4. 점주 WifiSettings 실 API 연동 (mock 제거)
5. mobile — 비콘 감지 → 매장 WiFi 묶음 fetch + 폰 캐시
6. mobile — 연결 후 검증(BSSID + 인증서 핀닝) 로직
7. mobile — "WiFi 정보 변경됨 [적용하기]" 흐름
8. mobile — OS 최소버전 게이트 (부팅 시 체크 + 차단 화면)
9. 손님 설정 — "전체 자동 / 매장별 승인" 선택지
10. `credential_mode` = static + managed 구현
11. 어드민 country 차원 + 권한 범위 (구조만, 우선 KR)
12. .mobileconfig 프로파일 생성 로직 (서명은 B 단계 인증서 후)
13. 비콘 9개 실기기 통합 리허설 (하드웨어 보유)

### B. 외부·행정 선행 후 (법인카드 이후)

- 사업자등록 → 통신판매업·위치기반서비스사업 신고
- Apple Developer($99) → 프로파일 서명 인증서, APNs, HotspotConfig
- 도메인 + SSL 인증서 → TLS 핀닝 최종 적용
- .mobileconfig 서명 적용
- 개인정보·위치정보 법률 전문가 검토

### C. 확장 (출시 후 / 이후)

- `credential_mode` = radius (WPA2-Enterprise + RADIUS) — 호텔·독서실 엄격 시간제 차단
- Eddystone-EID 회전 비콘 ID — 비콘 복제 원천 차단
- 예약시스템(PMS)·POS 연동 — 호실/주문 자동 수신, 카페 시간권 자동 발급
- 국가별 멀티 리전 인프라(AWS) + 국가별 약관·규제 패키지
- 글로벌 통합 어드민 검색·리포팅
